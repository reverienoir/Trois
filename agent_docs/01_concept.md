# 01. コンセプト / プロダクト哲学

## 課題（なぜ作るのか）

> お店がたくさんあって選ぶのが大変。悩んじゃうから、誰かに決めてほしい。

これがこのアプリの出発点。
既存のグルメサービス（食べログ・Google Maps 等）は **選択肢を増やす** 方向に最適化されている。
レビュー数千件、検索結果数百件 — 情報は豊富だが、その結果 **「決められない」** という別の苦痛を生む。
Trois はこの「決断疲れ」を解く。

## 価値と手段

普通のアプリは「価値」と「手段」が分離する。

| サービス | 価値 | 手段 |
|---|---|---|
| 食べログ | 店を探せる | レビュー多数を表示 |
| Google Maps | 目的地に着ける | 地図を表示 |
| **Trois** | **決めてもらえる** | **3軒だけ表示** |

Trois の特徴は、**「選択肢を減らすこと」自体が価値になっている** 点。
だから "3軒に絞る" は単なる仕様ではなく **サービス哲学＝ブランド宣言** になる。
「Trois なら3軒に絞ってくれる」という期待が、そのままブランドになる。

## なぜ "3" なのか（1でも10でもなく）

選択肢が多すぎると人は選べなくなる（選択過多）。これが解こうとしている課題。
一方、選択肢をゼロにして「ここに行け」と1軒だけ断定すると、今度は **主体性** を奪う。
「なんでこの店？自分で選んでない」という不信・不満につながりやすい。

**3軒は「ほぼ決めてもらえる安心」と「最後の一手は自分で選べる納得感」が両立するスイートスポット。**
よって "3" は恣意的な数ではなく、価値（決断補助）の最適な表現と位置づける。

## 名前の由来（Trois）

- フランス語で「3」。
- **3軒** 提案 ＋ 将来の **朝・昼・夜の3食** 対応、のダブルミーニング。
- 機能を足すほど名前の意味が強化される（"3食" 拡張で意味が増える）。
- 既存プロジェクト群（Reverie / Rue / Noir）と同じフランス語系で世界観が揃う。
- 意味が逐語で伝わる必要はない（Spotify 等と同じ）。意味はアプリ名ではなくサブタイトルが担う。
  - App Store サブタイトル案: 「AIが近くの3軒を提案 / 徒歩・予算で迷わず決まる」

---

## 守る / 可変（設計の羅針盤）

### 守る（不変）
- 決断疲れを減らす＝決めるのを肩代わりする
- AI が **理由つきで** 提案する
- "あえて絞る" 思想（凡庸な一覧表示にしない）

### 可変（実装詳細）
- 入力軸の追加（気分・時間帯・人数 など）
- 推薦理由の見せ方・UI 表現
- データソース・内部スコアリング手法

### 境界の注意
- 守るのは **"決断を減らす"** こと。
- **"必ず3件表示"** は現時点での最適な表現であって、永久不変のドグマではない。
  将来「今日はここ一択」型の断定もあり得る。ただし MVP は3軒固定でよい。

## コピー / ポジショニング（App Store 紹介文の種）

- 「店を探すアプリ、ではありません。**決めてくれる**アプリです。」
- 「悩んでる時間が、一番もったいない。」
- 「選択肢を増やすのではなく、迷いを減らすために。AIがあなたの代わりに考え、最適な3つを提案する。」
