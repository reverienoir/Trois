# 02. 画面設計メモ（MVP）

MVP は最小2画面（＋ローディング状態）。「入力 → 3軒提案」の一直線フローに絞る。

## 画面遷移

```
[HomeView 入力] --提案するタップ--> [LoadingView 取得・選定中] --完了--> [ResultView 3軒提案]
       ^                                                                    |
       |------------------------- もう一度 / 条件を変える ---------------------|
```

---

## 1. HomeView（入力画面）

ユーザーが条件を伝える画面。入力は **最小限・タップ中心**（モバイルで打たせない）。

要素:
- **現在地ステータス**: GPS取得中 / 取得済み（地名 or 「現在地」）/ 許可が必要、の表示
- **徒歩◯分以内**: セグメント or スライダー（例: 5分 / 10分 / 15分）
  - 内部では距離に換算（→ `03_data_model.md` の換算ルール）
- **予算**: 選択式（例: 〜1000円 / 〜2000円 / 〜3000円 / 上限なし）
- **ジャンル / 気分**: チップ選択（例: 中華 / あっさり / がっつり / 和食 / カフェ … 複数選択可も検討）
- **［提案する］ボタン**: これを押すと取得＋AI選定へ

設計メモ:
- 入力は「埋めなくても提案できる」ゆるさを許容（未指定はAI側で補完）。
- 位置情報の許可が無い場合の導線（設定アプリへの案内）を用意。

---

## 2. LoadingView（取得・選定中）

- 飲食店API取得 → Claude API で3軒選定、の待ち時間に表示。
- 「近くのお店を探しています」「3軒に絞っています」など、"絞る" 体験を演出するコピー。
- 体感を軽くするため、段階表示やスケルトンUIを検討。

---

## 3. ResultView（3軒提案画面）

このアプリの主役。**3軒のカードを縦に並べる**（横スワイプでもよいが、3枚なので一覧性重視）。

各カード要素:
- 店名
- ジャンル / 予算帯
- 現在地からの **徒歩◯分（距離）**
- **AIの推薦理由**（最重要。「雨だから駅近のここ」「予算内で評価が高い」等、なぜこの店かを一言）
- 店舗写真（APIが返す画像）
- アクション: 「地図で見る」（MapKit / 外部マップ）/「お店の詳細」（外部ページ）
  - ※ ホットペッパー利用時は **クレジット表記** を画面内に必須で入れる

画面下部:
- 「条件を変えてもう一度」/「3軒とも違う → 別の3軒を出す」導線

設計メモ:
- 3軒は **順位づけ or 役割づけ** を検討（例: 本命 / 穴場 / 無難）。"選びやすさ" が上がる。
- 推薦理由はAI生成テキスト。空・破綻時のフォールバック文言を用意。
