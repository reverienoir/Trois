# 05. マネタイズ設計

> ステータス: **実装済み（MVP）**

---

## プラン構成

| プラン | 回数 | 価格 | 1回あたり | Product ID |
|---|---|---|---|---|
| 無料 | 30回/月 | ¥0 | — | — |
| サブスク | 150回/月 | ¥500/月 | ¥3.33 | `com.eruru.Trois.monthly` |
| 追加購入 S | +10回 | ¥100 | ¥10.00 | `com.eruru.Trois.topup10` |
| 追加購入 L | +80回 | ¥400 | ¥5.00 | `com.eruru.Trois.topup80` |

- サブスク: 自動更新サブスクリプション
- 追加購入: 消費型IAP（期間リセット時に消える）
- Apple手数料: 15%（スモールデベロッパー初年度）/ 30%（通常）

---

## 使用回数ルール

- カウント期間: **課金日から30日**（暦月ではなく課金日起算）
- 30日経過で usageCount・extraUses・freeResetUsed をリセット
- 無料復活: **月1回・5回分**（freeResetUsed フラグで管理）
- 通知: 残り **10回・5回** のタイミングで通知

## Paywall 構成

上限到達時にシート表示。上から順に：

1. **追加購入 S**（10回 / ¥100）— 衝動買い層向け
2. **追加購入 L**（80回 / ¥400）— 「1回あたり半額」バッジ付き・ボーダー強調
3. **サブスク**（150回/月 / ¥500）— 「毎月たくさん使うなら」として別セクション
4. **無料復活**（5回 / 月1回）— 条件満たす時のみ表示

単価バーで S・L の1回あたりコストを視覚比較。
L → サブスク へ自然に誘導する設計。
