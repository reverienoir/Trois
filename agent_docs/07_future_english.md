# 07. 英語対応 / 訪日外国人ターゲット

> ステータス: **検討中（日本語版リリース後に着手）**

---

## なぜ英語対応するのか

訪日外国人は端末を英語設定のまま日本に来るケースが大半。
「近くのお店をAIが3軒に絞る」という価値は言語を問わず刺さる。
グルメアプリの英語対応は競合が少なく、差別化になりやすい。

---

## 方針

**端末の言語設定に自動追従**する。トップで言語選択画面は設けない。

- 理由: 起動時に選択を迫るのはコンセプト（迷わせない）と矛盾する
- 英語端末 → 英語UI、日本語端末 → 日本語UI が自然
- iOSの `Localizable.strings` を使ったローカライズで対応
- 設定画面から手動で言語変更できるようにする（サブ対応）

---

## 実装時の課題

### 1. UIテキストのローカライズ
- チップの文言（気分・シーン・ジャンル）は英語で再設計が必要
- 日本人向けの「疲れてる」「贅沢したい」をそのまま翻訳しても刺さらない
- 訪日外国人視点で再考する例:
  - "Quick bite" / "Local experience" / "No English menu OK" / "Vegetarian-friendly" など
  - シーン: "Solo" / "With colleagues" / "Date night" / "Family" はそのまま使える

### 2. ホットペッパーAPIのデータが日本語のまま
- 店名・ジャンル名は日本語で返ってくる
- ジャンル名は固定値なので英語への対応表（マッピング）を作れば対応可能
- 店名はそのまま表示（固有名詞なので翻訳不要・むしろ日本語のままが本物感）

### 3. Claude APIへのプロンプトを言語に応じて切り替える
- `Locale.current.language.languageCode` で判定し、プロンプトを英語版に切り替える
- AIの推薦理由・サマリーを英語で生成してもらう
- 比較的シンプルに対応できる見込み

### 4. 距離・予算の表示
- 徒歩分は "5 min walk" でそのまま通じる（変更不要）
- 予算は ¥ 表記のまま（円は訪日外国人にも馴染みがある）
