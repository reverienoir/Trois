# Trois デザイン発注書

## 1. プロダクト概要
Trois（トロワ）は、近くの飲食店をAIが「3軒だけ理由つきで」提案するiPhoneアプリ。

**コアバリュー**: 「店を探させる」のではなく「決断を肩代わりして、迷いを減らす」こと。
"あえて絞る" がこのプロダクトの信念であり、差別化の核。

### 守りたいこと（デザインでも壊さないでほしい価値）
- 決断負荷を減らす（選択肢を並べて選ばせるUIにしない）
- AIが理由つきで提案している、という"納得感"を視覚的にも伝える
- 食べログ／Google Maps的な「比較・検索」感を出さない（候補は3つだけ、という潔さ）

## 2. 現在の状態
- 機能（入力 → AI提案 → 結果表示）はひと通り動く MVP が完成
- 現状は素のSwiftUI標準コンポーネントのみで、デザインは未着手
- 今回お願いしたいのは「見た目・体験のデザイン」（情報設計や機能追加ではなく）

## 3. 画面構成（3画面 + ローディング）

### 3-1. ホーム画面（入力画面）
ユーザーが条件を入力する画面。上から順に：
1. ヘッダー: 「AIが近くの3軒を決めてくれる」
2. 場所: 現在地 / 地名で指定（セグメント切り替え＋テキスト入力）
3. 徒歩何分以内？: 5分 / 10分 / 15分 のチップ選択
4. 予算: 〜1000円 / 〜2000円 / 〜3000円 / 上限なし のチップ選択
5. 今の気分は？（複数選択可）: 疲れてる／元気を出したい／健康的に／がっつり／あっさり／贅沢したい／冒険したい／失敗したくない
6. シーン（単一選択）: 一人で／同僚と／デート／家族と
7. ジャンルで絞る（任意・複数可）: ラーメン／焼肉／カレー／そば・うどん／居酒屋／ファストフード／和食／中華／カフェ
8. 自由入力欄（プレースホルダー例: 「さっぱりしたものが食べたい」）
9. 「3軒を提案してもらう」ボタン

すべての入力は任意（最低限「場所」が取得できればOK）。

### 3-2. ローディング画面
提案生成中に表示。複数のメッセージが順番に切り替わる：
「近くのお店を探しています…」→「候補を集めています…」→「3軒に絞っています…」→「推薦理由を考えています…」

### 3-3. 結果画面
- 上部: AIが「なぜこの3軒を選んだか」の全体サマリー（1〜2文）
- 3軒分のカード（店舗写真・店名・ジャンル/予算/徒歩分・役割バッジ「本命/穴場/無難」・AIの推薦理由・「地図で見る」「お店の詳細」リンク）
- （3軒に満たない場合）お詫びメッセージのバナー
- フッター: 「Powered by ホットペッパー Webサービス」表記（必須）
- 「条件を変えてもう一度」ボタン

## 4. デザインしてほしいこと
- 全体のトンマナ（色・フォント・余白などのビジュアルアイデンティティ）
- 入力画面のチップ/トグル類のスタイル（複数カテゴリが並ぶので情報の見やすさ・押しやすさ重視）
- ローディング画面の演出（"AIが考えている感"を出す。単なるスピナーではなく独自性がほしい）
- 結果画面のカードデザイン（写真・バッジ・推薦理由が映える、かつ"3軒だけ"という特別感が出るレイアウト）
- 「本命/穴場/無難」の3つの役割を視覚的に区別する見せ方（バッジ・色など）
- アイコン/イラスト（あれば）
- ダークモード対応の有無の方針

## 5. トーン＆マナーの方向性（叩き台。ここも一緒に相談したい）
- 「迷いを減らす」「肩代わりしてくれる」という安心感・信頼感
- AIが"友人として提案してくれる"ような温かみ
- ギラギラした派手さよりは、シンプルで余白のある落ち着いたデザイン
- ※決め打ちではないので、ClaudeDesign側からの提案も歓迎

## 6. 技術的制約
- iOS / SwiftUI実装（Swift Viewへの落とし込みがしやすい形でのアウトプットが望ましい）
- 既存のチップ選択コンポーネント（`ChipToggleStyle`、`FlowLayout`）があるため、デザインに合わせて改修する想定
- ホットペッパーWebサービスのクレジット表記は変更不可（デザインへの組み込み方は相談したい）
